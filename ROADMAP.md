# Roadmap

This roadmap describes likely direction, not a binding delivery promise.

## Now

- Stabilize the first public CLI release.
- Keep conventional commit parsing predictable and fixture-backed.
- Document release-note workflows clearly for local and agentic use.

## Next

- Add richer examples for monorepos and scoped package releases.
- Improve template placeholders if real release drafts need more structure.
- Add CI recipes for generating review-only changelog artifacts.

## Later

- Consider SARIF or other structured outputs if release systems need them.
- Consider changelog section customization once the default grouping is stable.
- Consider optional GitHub release drafting, but only as an explicit non-default
  integration.

## Not Planned

- Publishing releases automatically.
- Creating git tags or rewriting git history.
- Replacing commitlint or semantic-release.
